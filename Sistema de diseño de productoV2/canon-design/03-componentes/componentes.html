Biblioteca de componentes

Los componentes de la aplicación densa, con sus estados. El oro es escaso e intencional: aquí solo aparece donde significa valor, acción o evidencia verificada. La confianza se comunica con oro + grises + forma + texto — nunca con color solo, nunca con semáforo.

★

Niveles de evidencia

El nivel no es un número que el LLM se auto-reporta (eso sería el "ATS score" que condenamos). Deriva de evidencia real: verificado = el fragmento aparece literal en la fuente, o es dato de API, o lo escribió el humano · parcial = coincidencia difusa · sin verificar = sin evidencia rastreable, y la UI lo señala. Regla: la ausencia de oro también comunica. Segunda señal siempre (forma + texto), nunca solo color.

Verificado
Verificado

Oro presente. Rombo lleno ◆. El evidenceSnippet aparece literal en la fuente, o viene de una API, o lo escribió el humano.

Parcial
Parcial

Oro tenue. Rombo medio ◈. Coincidencia difusa: el fragmento no calza exacto. Merece una mirada antes de aceptar.

Sin verificar
Sin verificar

Cero oro, borde punteado, círculo hueco ○. Sin evidencia rastreable. La UI lo señala; nunca se presenta como hecho.

Toggle global de IA · y el modo con IA apagada

Ninguna función core exige una llamada al LLM. Crear el master, editar, componer variantes, exportar el PDF, ver "cómo lo lee el ATS" y el chequeo de salud funcionan sin IA. La IA acelera la entrada de datos y sugiere adaptaciones. Nada más.
